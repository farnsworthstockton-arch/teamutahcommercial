# 📥 Upload Your Files Here

This is your single source of truth for everything related to designations, certifications, courses, and awards.

---

## Quick Drop (lazy mode)

Don't know where something goes? Just dump it in:

```
designations/INBOX/
```

…and tell Claude "sort the INBOX" — files get auto-moved to the right place.

---

## Organized Drop (preferred)

Every credential has its own folder under `designations/items/<CODE>/` with four subfolders:

```
designations/items/<CODE>/
├── certificate/    ← The official certificate PDF (one per credential)
├── logo/           ← Designation badge or logo image (PNG/JPG/SVG)
├── documents/      ← Course materials: syllabus, slides, handouts, workbooks
└── notes/          ← Your personal notes, summaries, exam-prep
```

### Naming convention

Lead the filename with the code, e.g.:

- `CREN_certificate.pdf`
- `CREN_badge.png`
- `CREN_syllabus.pdf`
- `CREN_notes.md`

## Updating Status When You Finish a Course

From the `designations/` folder, run:

```powershell
python tracker.py set CREN status enrolled
python tracker.py set CREN enrolled_date 2026-06-01

# When done:
python tracker.py set CREN status completed
python tracker.py set CREN completed_date 2026-07-15

# Then refresh the website data:
python tracker.py export
```

…or just ask Claude: *"I finished CREN today, update everything."*

---

## What Goes On the Website

The bio page (`stockton-draft.html`) and the per-credential subpages (`designation.html?code=<CODE>`) automatically read from:

- `designations/designations.json` (status + dates, regenerated by `tracker.py export`)
- `designations/pages-content.json` (overview, topics, "how this serves clients")

Files in `items/<CODE>/certificate/` are auto-linked from the subpage when `cert_path` is set in the DB.

To pre-fill or edit a subpage's narrative content, edit `designations/pages-content.json`.
